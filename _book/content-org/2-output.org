** Output 

配置 =output= 属性就是告诉webpack如何写编译好的文件输出到磁盘.注意,尽管可以设置多个 entry point,但是仅可以有一个 ouput 被指定.

*** Usage 

最小配置如下:

#+BEGIN_SRC js
  module.exports = {
    output: {
      filename: 'bundle.js',
    }
  };
#+END_SRC

*** Multiple Entry Points

如果你配置创建了多个 =chunk= (因为携带了多个 entry point 或者当你使用 plugins ,比如 CommonsChunkPlugin的时候).

你应该使用 =substitution= (替换) 来确保每个文件都有一个独一无二的名字.

#+BEGIN_SRC js
  module.exports = {
    entry: {
      app: './src/app.js',
      search: './src/search.js'
    },
    output: {
      filename: '[name].js',
      path: __dirname + '/dist'
    }
  };

  // writes to disk: ./dist/app.js, ./dist/search.js
#+END_SRC

*** Advanced

下面是对 assets使用CND 和 hashes的例子.

#+BEGIN_SRC js
 config.js
 ------------------------------
 module.exports = {
   //...
   output: {
     path: '/home/proj/cdn/assets/[hash]',
     publicPath: 'https://cdn.example.com/assets/[hash]/'
   }
 };
#+END_SRC

#+BEGIN_SRC
 __webpack_public_path__ = myRuntimePublicPath;

#+END_SRC
